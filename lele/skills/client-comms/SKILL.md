## Engagement hook

End every client-facing reply that leaves something open with a **single specific follow-up question or offer**. Never close a client message with nothing actionable for them.

**Patterns that generate replies (from log.md analysis):**
- Concrete deliverable + specific follow-up ask: "Here's the 4-week activation scope — want me to write the message templates, or map the n8n sequence first?"
- Status update that surfaces a decision: "The Calendly flow is ready to build — still need message copy from you. Here's a draft if that helps."
- Proactive blocker flag: "Waiting on message copy for the welcome DM. Quick reminder — anything close to this work?"
- Short question with ≤2 options: "For the Tolt group workflow: one-time migration or recurring? Let me know and I'll build it today."

**What to avoid:**
- Walls of text or structured A/B/C frameworks (overwhelming)
- Messaging clients about internal/technical issues
- Sending a second unprompted message if the first got no reply
