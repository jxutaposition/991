# n8n — Integration Requirements

## Credentials

n8n API key — configured in Settings > Integrations.
Also requires: n8n base URL (the instance URL, e.g., `https://n8n.example.com`).

## Access Model

- The API key gives full access to the n8n instance: workflow CRUD, execution, credential listing, activation/deactivation.
- **n8n has its OWN credential store** for third-party services (Slack, Notion, Google Sheets, etc.). These are separate from Lele's credentials.
- Check available n8n credentials with `GET {base_url}/api/v1/credentials`.

## Runtime Configuration

### Downstream Service Credentials (within n8n)

- **What**: n8n workflows that connect to Slack, Notion, or other services need credentials configured INSIDE n8n's own credential store.
- **Why**: Lele's Slack token is NOT the same as n8n's Slack token. They are completely separate credential stores.
- **How to check**: Call `GET {base_url}/api/v1/credentials` and look for the required credential types (e.g., `slackApi`, `notionApi`).
- **If missing**: Use `request_user_action` to instruct the user to add the credential in the n8n UI. This often requires an OAuth flow that can only happen in n8n's browser interface:
  ```json
  {
    "type": "steps",
    "title": "Add Slack credentials to n8n",
    "steps": [
      {"step": 1, "label": "Open your n8n instance", "detail": "Go to {base_url}"},
      {"step": 2, "label": "Navigate to Credentials", "detail": "Settings → Credentials → Add Credential"},
      {"step": 3, "label": "Add Slack API credential", "detail": "Select 'Slack API' and complete the OAuth flow"},
      {"step": 4, "label": "Reply here when done", "detail": "Type 'done' so I can verify the credential exists"}
    ]
  }
  ```

### Slack Channel (when workflow posts to Slack)

- **What**: Channel ID for Slack notifications within the n8n workflow
- **Input type**: `slack_channel`
- **How to ask**: Same as the Slack tool — use a typed input in `request_user_action`
- **Note**: The Slack credential in n8n must have access to this channel

### Notion Database (when workflow creates Notion pages)

- **What**: Database ID for page creation within the n8n workflow
- **Input type**: `notion_database`
- **How to ask**: Same as the Notion tool — use a typed input in `request_user_action`
- **Note**: The Notion credential in n8n must have access to this database

### Webhook URLs (from other systems)

- **What**: When building a workflow triggered by a webhook (e.g., Clay fires a webhook to n8n), the webhook URL is generated by n8n when the workflow is created.
- **Resolution**: This is NOT user input — the agent creates the workflow, gets the webhook URL from n8n, and passes it to the upstream system (e.g., Clay action column). This is a dependency ordering issue, not a configuration gap.
